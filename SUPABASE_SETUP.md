# Supabase Database Setup

## Required Tables

Your frontend is now connected to fetch real data from Supabase. Create these tables:

### 1. Products Table

Use your existing table schema:

```sql
create table public.products (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  last_update timestamp with time zone null default now(),
  brand text null,
  category text null,
  avg_w_p integer null default 0,
  avg_r_p integer null default 0,
  availability smallint null default '0'::smallint,
  registration_status smallint null default '0'::smallint,
  description text not null,
  country text null,
  constraint products_pkey primary key (id)
) TABLESPACE pg_default;
```

**Field Mapping:**
- `avg_w_p` → wholesale price (integer)
- `avg_r_p` → retail price (integer)
- `availability` → 0 = 'Out of stock', 1 = 'In stock', 2 = 'Limited'
- `registration_status` → 0 = not registered, 1 = registered
- `description` → used for both name and full description
- `last_update` → last update timestamp

### 2. User Entitlements Table (Optional)

Since this table doesn't exist in your database, the app uses user metadata for entitlements. If you want to add premium user management:

```sql
create table public.user_entitlements (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id) on delete cascade,
  plan text default 'free',
  premium_access boolean default false,
  next_billing_date timestamp with time zone,
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now(),
  unique(user_id)
);

-- Add index for user lookups
create index user_entitlements_user_id_idx on user_entitlements(user_id);
```

### 3. Premium Interest Table

Use this to record users who clicked pay and want to be notified when premium is ready:

```sql
create table public.premium_interest (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id) on delete cascade,
  email text,
  company_name text,
  company_type text,
  status text not null default 'ready_to_pay',
  created_at timestamp with time zone not null default now(),
  updated_at timestamp with time zone not null default now(),
  unique(user_id)
);

alter table public.premium_interest enable row level security;

create policy "Users can insert their own premium interest"
  on public.premium_interest for insert
  to authenticated
  with check (auth.uid() = user_id);

create policy "Users can update their own premium interest"
  on public.premium_interest for update
  to authenticated
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);
```

**Note:** Currently, premium access is determined by the `premium_access` field in user metadata set during signup.

## Frontend API Functions

The frontend now uses these functions to fetch data:

### `getProducts(params)`
- Queries the products table directly from Supabase
- Supports filters: `category`, `brand`, `registered`, `search`
- Falls back to mock data if the query fails
- Example:
```javascript
getProducts({ 
  category: 'Industrial',
  search: 'FieldGuard'
})
```

### `getEntitlements()`
- Fetches the current user's entitlement status from `user_entitlements`
- Uses the authenticated user's ID
- Falls back to checking `user_metadata.premium_access` if table is unavailable
- Falls back to free tier if query fails

## Column Mapping

The frontend expects camelCase but the database uses snake_case. The API layer automatically maps:
- `short_description` → `shortDescription`
- `wholesale_price` → `wholesalePrice`
- `retail_price` → `retailPrice`
- `last_update` → `lastUpdate`
- `premium_access` → `premiumAccess`
- `next_billing_date` → `nextBilling`

## Row Level Security (RLS)

Make these tables readable by authenticated users:

```sql
alter table public.products enable row level security;

create policy "Products are readable by authenticated users"
  on public.products for select
  to authenticated
  using (true);

alter table public.user_entitlements enable row level security;

create policy "Users can read their own entitlements"
  on public.user_entitlements for select
  to authenticated
  using (auth.uid() = user_id);
```
